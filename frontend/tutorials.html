    Getting Started

    You can freely explore the entire platform without needing to register or log in. The Model Catalog is accessible to all visitors, allowing you to browse a wide range of Earth Observation (EO) machine learning models. You can view detailed information about each model, including metadata, supported tasks, and download links for sample datasets.

    To upload your own models and contribute to the community, you need to create an account. Register via the Registration page and log in to access the upload functionality. After logging in, you can also track your uploaded models and manage your contributions through your personalized Dashboard.

    About the Website

    The HuggingEarth platform serves as a centralized hub for discovering, sharing, and using machine learning models designed for geospatial and EO data. It enables users to find models for tasks like classification, segmentation, and prediction, streamlining workflows for research, monitoring, and data analysis.

    What is STAC?

    The SpatioTemporal Asset Catalog (STAC) is a standardized format for describing geospatial assets, ensuring consistency and interoperability between datasets. Our models and metadata comply with the STAC specification, allowing seamless integration with geospatial tools and libraries like pystac.

    Example STAC Collection

    This example shows how a STAC-compliant ML model collection can be structured:

    {
  "stac_version": "1.0.0",
  "stac_extensions": [
    "https://stac-extensions.github.io/item-assets/v1.0.0/schema.json"
  ],
  "type": "Collection",
  "id": "ml-model-examples",
  "title": "Machine Learning Model examples",
  "description": "Collection of items contained in the Machine Learning Model examples.",
  "license": "Apache-2.0",
  "extent": {
    "spatial": {
      "bbox": [
        [
          -7.882190080512502,
          37.13739173208318,
          27.911651652899923,
          58.21798141355221
        ]
      ]
    },
    "temporal": {
      "interval": [
        [
          "1900-01-01T00:00:00Z",
          "9999-12-31T23:59:59Z"
        ]
      ]
    }
  },
  "item_assets": {
    "weights": {
      "title": "model weights",
      "roles": [
        "mlm:model",
        "mlm:weights"
      ]
    }
  },
  "summaries": {
    "datetime": {
      "minimum": "1900-01-01T00:00:00Z",
      "maximum": "9999-12-31T23:59:59Z"
    }
  },
  "links": [
    {
      "href": "collection.json",
      "rel": "self"
    },
    {
      "href": "item_basic.json",
      "rel": "item"
    },
    {
      "href": "item_bands_expression.json",
      "rel": "item"
    },
    {
      "href": "item_eo_bands.json",
      "rel": "item"
    },
    {
      "href": "item_eo_and_raster_bands.json",
      "rel": "item"
    },
    {
      "href": "item_eo_bands_summarized.json",
      "rel": "item"
    },
    {
      "href": "item_raster_bands.json",
      "rel": "item"
    },
    {
      "href": "item_multi_io.json",
      "rel": "item"
    }
  ]
}

    Explanation of the Key Fields:

    • stac_version: The version of the STAC specification, here "1.0.0".
    • stac_extensions: A list of STAC extensions that are relevant to the model (e.g., EO for Earth Observation, ML for Machine Learning, etc.).
    • type: The type of the STAC resource, in this case, a "Feature".
    • id: A unique ID for the model.
    • properties: A collection of key-value pairs for metadata about the model:
      • title and description describe the model.
      • start_datetime and end_datetime define the period during which the model is trained or available.
      • model_task specifies the task of the model (e.g., classification, regression).
      • input_data_type, output_data_type, input_bands, and output_classes specify the types of input and output data, as well as the input bands and output classes of the model.
    • geometry: The spatial extent of the model, here specified as a Polygon.
    • links: URLs that link to other resources or references related to the model (e.g., the model itself, thumbnails, etc.).
    • assets: Files associated with the model, such as the model file itself and metadata files.

    Customization:

    • id: Customize the model ID.
    • title, description, model_task, etc.: These fields should be adjusted to reflect the actual model.
    • assets: The links to model files and metadata should point to the actual files on your server or in your repository.

    Integrating with Python: A Step-by-Step Guide

    Easily integrate machine learning models into your Python workflows by following these steps:

    1. Install Python: Ensure you have Python installed on your system. You can download the latest version of Python here.
    2. Install the required STAC client: Use pip install pystac for Python to set up the necessary tools.
    3. Download model metadata: Navigate to the Catalog page to select and download the required metadata.
    4. Integrate the model: Use pystac.Item() to load and manipulate the downloaded metadata for your workflows.

    Integrating with R: A Step-by-Step Guide

    Easily integrate machine learning models into your R workflows by following these steps:

    1. Install R: Ensure you have R installed on your system. You can download the latest version of R here.
    2. Install the required STAC client: Use install.packages('rstac') for R to set up the necessary tools.
    3. Download model metadata: Navigate to the Catalog page to select and download the required metadata.
    4. Integrate the model: Use stac_read() to load and manipulate the downloaded metadata for your workflows.
