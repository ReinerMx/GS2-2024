    Uploading Your Own Models: A Quick Guide

    Easily share your machine learning models with the community by following these steps. Ensure your metadata complies with the STAC standard for seamless integration.

    1. Prepare your metadata: Create your metadata in STAC-compliant JSON format using this template.
    2. Upload your file: Navigate to the Upload page, complete the form, and upload your file.
    3. Handle errors: If you encounter any issues during the upload process, a detailed error message will appear. For further assistance, feel free to Contact Us.
    4. Access your models: Successfully uploaded models will be visible in your account.

    Explanation of the Key Fields:

    • stac_version: The version of the STAC specification, here "1.0.0".
    • stac_extensions: A list of STAC extensions that are relevant to the model (e.g., EO for Earth Observation, ML for Machine Learning, etc.).
    • type: The type of the STAC resource, in this case, a "Feature".
    • id: A unique ID for the model.
    • properties: A collection of key-value pairs for metadata about the model:
      • title and description describe the model.
      • start_datetime and end_datetime define the period during which the model is trained or available.
      • model_task specifies the task of the model (e.g., classification, regression).
      • input_data_type, output_data_type, input_bands, and output_classes specify the types of input and output data, as well as the input bands and output classes of the model.
    • geometry: The spatial extent of the model, here specified as a Polygon.
    • links: URLs that link to other resources or references related to the model (e.g., the model itself, thumbnails, etc.).
    • assets: Files associated with the model, such as the model file itself and metadata files.

    Customization:

    • id: Customize the model ID.
    • title, description, model_task, etc.: These fields should be adjusted to reflect the actual model.
    • assets: The links to model files and metadata should point to the actual files on your server or in your repository.

    STAC-Conform JSON-Template:

    Ensure your JSON file follows this structure:

      {
    "stac_version": "1.0.0",
    "stac_extensions": [
      "https://stac-extensions.github.io/eo/v1.0.0/schema.json",
      "https://stac-extensions.github.io/label/v1.0.0/schema.json",
      "https://stac-extensions.github.io/ml/v1.0.0/schema.json"
    ],
    "type": "Feature",
    "id": "example-model-id",
    "properties": {
      "title": "Example ML Model for Land Classification",
      "description": "A machine learning model for classifying land use types based on satellite imagery.",
      "start_datetime": "2023-01-01T00:00:00Z",
      "end_datetime": "2023-12-31T23:59:59Z",
      "model_task": "classification",
      "input_data_type": "satellite imagery",
      "output_data_type": "land use categories",
      "input_bands": [
        "red",
        "green",
        "blue",
        "infrared"
      ],
      "output_classes": [
        "forest",
        "urban",
        "water",
        "agriculture"
      ],
      "cloud_coverage": 5,
      "platform": "Sentinel-2",
      "version": "v1.0"
    },
    "geometry": {
      "type": "Polygon",
      "coordinates": [
        [
          [-180.0, -90.0],
          [180.0, -90.0],
          [180.0, 90.0],
          [-180.0, 90.0],
          [-180.0, -90.0]
        ]
      ]
    },
    "links": [
      {
        "rel": "self",
        "href": "https://example.com/stac/catalog/example-model-id"
      },
      {
        "rel": "item",
        "href": "https://example.com/models/example-model-id"
      },
      {
        "rel": "thumbnail",
        "href": "https://example.com/thumbnails/example-thumbnail.jpg"
      }
    ],
    "assets": {
      "model_file": {
        "href": "https://example.com/models/example-model.onnx",
        "title": "Model File",
        "type": "application/onnx"
      },
      "metadata": {
        "href": "https://example.com/models/example-model-metadata.json",
        "title": "Metadata File",
        "type": "application/json"
      }
    }
  }

    Integrating with Python and R: A Step-by-Step Guide

    Easily integrate machine learning models into your Python or R workflows by following these steps:

    1. Install Python or R: Ensure you have Python or R installed on your system. You can download the latest version of Python here or R here.
    2. Install the required STAC client: Use pip install pystac for Python or install.packages('rstac') for R to set up the necessary tools.
    3. Download model metadata: Navigate to the Catalog page to select and download the required metadata.
    4. Integrate the model: Use pystac.Item() in Python or stac_read() in R to load and manipulate the downloaded metadata for your workflows.

    General

    The HuggingEarth-Catalog is a platform that provides geoscientific models to browse, retrieve, and integrate data into workflows.

    STAC (SpatioTemporal Asset Catalog) is a standard for organizing geospatial data, ensuring consistency and interoperability between datasets. Learn more on the official STAC website.

    Account

    Click on "Login" on the Login page and follow the instructions. A valid email address is required for registration. Make sure to use a secure password.

    Please Contact Us. Currently we can only reset your password this way.

    Models

    The platform offers various models for climate data analysis, environmental monitoring, and geodata integration. You can find a list of models on the Catalog page.

    For large models, ensure your internet connection is stable. If the upload fails, try compressing your model into a ZIP file before uploading.

    Technical

    Install the `pystac` library and use it to integrate data directly from the platform into your workflows. Example codes can be found in our PySTAC tutorial.

    Use our metadata validation tool on the Upload page. Ensure your file is in STAC-compliant JSON format.
